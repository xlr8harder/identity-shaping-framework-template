# Identity Shaping Agent Guide

Coding agents are the primary users of this repository. Begin by reading
`docs/README.md`, `docs/setup.md`, `docs/workflow.md`, `identity/SEED.md`, and
`isf.yaml`. Inspect `isf --help` and the relevant subcommand help before
changing project structure or inventing new commands.

First report the current project state, missing prerequisites, assumptions, and
the next concrete identity-development step. Do not start paid inference, large
data-generation runs, or training until their inputs and configuration have
been validated with the user.

Follow the repository's documented phases and preserve its conventions. Sample
and inspect generated outputs before expanding a run, and do not train on data
that has not been validated.
